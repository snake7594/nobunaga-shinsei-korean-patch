# 대사 줄바꿈(\n) 처리 — msgev(나레이션)만 제거, msggame(캐릭터 대사)는 보존

## 게임의 대사 렌더링 (두 가지 박스)
- **전체폭 시나리오/이벤트 나레이션 박스** = `msgev.bin`. 폭(~1150px)을 넘으면 **자동 줄바꿈**.
  → 수동 `\n`을 제거하면 엔진이 폭에 맞춰 깨끗이 재줄바꿈한다(오버플로 해소). **제거가 옳다.**
- **이름표/초상화 캐릭터 대사 박스** = `msggame.bin`. 자동 줄바꿈이 아니라 **한 줄을 폭에 맞춰
  글자 크기를 축소**. → `\n`을 제거해 한 줄로 합치면 글자가 지나치게 작아진다. **건드리면 안 된다.**

## 근거 (msgev=전체폭, msggame=축소 박스)
- 이벤트 스크립트 `FLOW_PK/*.bsf`는 컷신 텍스트를 화자=99998(=이름표 없음, 전체폭)로 표시한다.
  OPENING에서는 의미상 "대사"인 문장(예: 毛利元就의 발화)도 전부 99998 전체폭이었다.
- JP 원문 줄 폭(폰트 advance) 분포: **msgev는 모든 인덱스 구간에서 p50≈864~912px**(전체폭
  나레이션과 동일), **msggame는 p50=528px**(확연히 좁은 이름표 박스). → msgev에는 축소 박스가
  없다. `tools/box_width_probe.py` 참조.
- 사고 이력: 2026-07-23 `strip_dialogue_nl.py`로 msgev+msggame 전체(~2.1만개) `\n`을 제거했다가
  msggame 캐릭터 대사가 축소돼 반려 → 전면 롤백(커밋 b35d761). 원인은 **msggame까지 건드린 것**.

## 현재 방침 (2026-07-24, 커밋 이후)
- **`strip_msgev_only.py`** 로 **msgev만** `\n` 제거(6,674건), **msggame은 손대지 않는다**(원본 `\n` 유지).
- 서식 문자열(【】●※·불릿·정렬 표·토글·제어 글리프)은 분류기로 자동 제외.
- **운문/명문·단어중간 절단 7건은 `protect_msgev.json`으로 내용 기준 보호**(버전 간 인덱스가
  달라 내용 매칭). 사세구·한시·와카·칼 명문 6건(고유 들여쓰기가 행 경계·서명 정렬 담당) +
  단어 중간을 끊는 `\n` 1건. 16-청크 적대적 검증 워크플로로 확정.
- `puk_msgev_fix(_117).py`(세그먼트>1000만 제거하던 보수적 처리)는 이 방식으로 대체됨.

## 빌드 순서
`build_msgpk(_117).py` → `msgpk_normalize(_117).py` → `strip_msgev_only.py`
(`PROTECT=tools/protect_msgev.json`, `ROOT=<빌드의 MSG_PK/JP>`). msggame은 build 단계 원본 `\n` 유지.
