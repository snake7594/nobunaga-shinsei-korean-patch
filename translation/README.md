# 번역 데이터

노부나가의 야망 신생(信長の野望・新生, 버전 1.1.5/파워업키트) 한글 번역 데이터입니다.

- `source_jp/`  — 일반 게임(872000) 원문 문자열 배치 (번역 키). 529개 파일.
- `korean/`     — index로 짝지어진 한국어 번역. 529개 파일.
- `source_jp_puk/` — **파워업키트(872001) 신규** 원문 문자열 배치. 45개 파일.
  (일반 게임과 겹치는 문자열은 `source_jp/`+`korean/` 사전을 그대로 재사용하므로
  여기 있는 건 파워업키트에만 있는 시나리오·정책·전법·튜토리얼 등 신규 콘텐츠뿐입니다.)
- `korean_puk/`    — index로 짝지어진 한국어 번역. 45개 파일.

## 파일 형식

원문 `source_jp/bXXXX_*.json` / `source_jp_puk/pkNNN.json`:
```json
{ "cat": "dialog",
  "items": [ { "i": 0, "t": "원문 문자열" }, ... ] }
```
(`source_jp_puk/`는 `cat` 필드 없이 `items`만 있습니다.)

번역 `korean/bXXXX_*.json` / `korean_puk/pkNNN.json`:
```json
{ "items": [ { "i": 0, "t": "번역 문자열" }, ... ] }
```

- `i` 인덱스로 원문↔번역이 대응됩니다(같은 파일명끼리 짝지어짐).
- 서식 토큰은 보존해야 합니다: `%s %d`, `\n`(줄바꿈), `[bNNN]`류 이름 플레이스홀더,
  `<ESC>C?`(U+001B) 컬러 태그.
- 적용은 `tools/apply_translations.py`(일반 게임) / `tools/build_msgpk.py`(파워업키트)가
  원문 문자열을 키로 매칭해 MSG 파일에 주입합니다.
- `korean_puk/`에는 시나리오 검색용 순수 가나 문자열(예: `カワナカジマノタタカイマエ`)이
  원문과 동일하게(=미번역 상태로) 들어있는 항목이 소수 있습니다. 의도된 것입니다 —
  자세한 이유는 [docs/FORMATS.md §7](../docs/FORMATS.md#7-파워업키트872001-이중-프로그램-구조) 참고.

## 품질

- 기계 번역이 상당수 포함되어 있으며 일부 오역·어색함이 있을 수 있습니다.
- 인명·지명은 국립국어원 일본어 표기법을 지향합니다.
- 개선/기여 환영합니다. `korean/` 의 `t` 값을 고쳐 PR 하시면 됩니다.

## 고지

- 팬 번역이며 코에이 테크모와 무관합니다.
- 원문(`source_jp/`)은 번역 작업용 참조 키로, 게임 원본 데이터·에셋은 포함하지 않습니다.
